# MOMO Project: MOMO Explorer 実装仕様書

## 1. プロジェクト概要
「MOMO Project」に属する各Webアプリが利用する3種のストレージバックエンド（momo_gdrive / momo_disk / momo_github）を、Windowsエクスプローラに近い操作感で統合的に閲覧・操作するためのWebアプリケーションを構築する。
独立したHTMLページとしてGitHub Pages上に公開し、Pyodide（ブラウザ内Python実行環境）を埋め込むことで、3本のPythonライブラリをそのままブラウザ上から呼び出せる構成とする。
ユーザー向けの汎用ファイルマネージャーとして設計するが、3本ライブラリの統合動作テストベッドとしての役割も兼ねる。

## 2. ファイル配置
- 公開URL: `https://qiqiroon.github.io/momo/tools/explorer/`
- 配置パス（リポジトリ内）: `momo/tools/explorer/index.html`
- 依存ライブラリ（リポジトリ内）:
  - `momo/lib/momo_gdrive/momo_gdrive.py`
  - `momo/lib/momo_disk/momo_disk.py`
  - `momo/lib/momo_github/momo_github.py`

## 3. 技術仕様
- 実行環境: ブラウザ（Chrome / Edge 推奨）
- Python実行: Pyodide 0.2x 以上（index.html に埋め込み）
- UIレンダリング: HTML / CSS / JavaScript（単一ファイル構成）
- バックエンド: momo_gdrive / momo_disk / momo_github（Pyodide経由で呼び出し）
- ファイルシステムアクセス: File System Access API（momo_disk 経由）
- ブラウザ制約: File System Access API の制約上、momo_disk は Chrome / Edge のみ動作
- JS ブリッジ: `momo_disk_bridge.js` を起動時に CDN（jsDelivr）から動的ロードする

## 4. 画面構成

### 4.1 全体レイアウト
```
┌─────────────────────────────────────────────────────┐
│ 🐱 MOMO Explorer ver X.XX                           │  ← ヘッダー
├─────────────────────────────────────────────────────│
│ 📁 gdrive/momo/tools/calc          [アドレスバー]    │
├─────────────────────────────────────────────────────│
│ [コピー] [切り取り] [貼り付け] [削除] [新規フォルダ] │  ← ツールバー
├──────────────────┬──────────────────────────────────│
│                  │                                  │
│  左ペイン        │  右ペイン                        │
│  フォルダツリー  │  フォルダ内容リスト              │
│                  │                                  │
│                  │                                  │
└──────────────────┴──────────────────────────────────┘
```

### 4.2 ヘッダー
- 左上に猫のアイコンと `MOMO Explorer ver X.XX` を小さく表示する。
- バージョン番号はHTMLソース内の定数として管理し、リリース時に更新する。

### 4.3 アドレスバー
- 現在選択中のフォルダのフルパスを表示する。
- パス表記の形式: `{バックエンド名}/{パス}` （例: `gdrive/momo/tools/calc`、`disk/music/2024`、`github/data/maps`）
- テキストとして直接編集可能とし、Enterキーで指定パスに直接ジャンプする。
- 存在しないパスを入力した場合はエラーメッセージをアドレスバー下に表示する。
- バックエンド名のプレフィックス（`gdrive/` 等）でどのバックエンドにアクセスするかを判定する。
- disk バックエンドへのジャンプは接続済みの場合のみ有効。未接続の場合はエラーメッセージを表示する。
- gdrive バックエンドへのジャンプは現在未実装（エラーメッセージを表示）。
- バックエンドに応じてアドレスバーのテキスト色を変更する（github: 緑、disk: 黄、gdrive: 赤）。

### 4.4 ツールバー
アドレスバーの下に以下のボタンを横並びで配置する。
- **コピー**: 選択中のファイル・フォルダをクリップボード（内部状態）に記録する。
- **切り取り**: 選択中のファイル・フォルダを切り取り対象として記録する。github バックエンドのファイルを対象とした場合はエラー表示。
- **貼り付け**: 現在右ペインに表示中のフォルダに貼り付ける。コピー・切り取りの記録がない場合はボタンを非活性とする。
- **削除**: 選択中のファイル・フォルダを削除する。github バックエンドのファイルを対象とした場合はエラー表示。
- **新規フォルダ**: 現在右ペインに表示中のフォルダ内に新規フォルダを作成する。フォルダ名入力ダイアログを表示する。github バックエンドを対象とした場合はエラー表示。

操作対象ファイルが未選択の場合、コピー・切り取り・削除ボタンは非活性とする。

### 4.5 左ペイン（フォルダツリー）
- ツリーのルートノードとして `github` / `disk` / `gdrive` の3項目を固定表示する。
- 各ルートノードは展開・折りたたみが可能で、展開するとそれぞれのバックエンドのフォルダ階層をツリー表示する。
- github のルートノードを展開した際に表示するトップフォルダは `data/` のみとする（リポジトリ全体は表示しない）。
- disk のルートノードを展開する際、未初期化の場合は `momo_disk.py` を GitHub から fetch して Pyodide に読み込み、`MomoDisk()` インスタンスを生成する。その後 `open_root()` を呼び出してディレクトリ選択ダイアログを表示する。2回目以降は前回接続を自動復元する。
- gdrive のルートノードをクリックした場合は「現在実装中」のダイアログを表示する（momo_gdrive 実装待ち）。
- フォルダをクリックすると、そのフォルダが選択状態になり、右ペインに内容を表示する。アドレスバーも同期して更新する。
- 現在選択中のフォルダはハイライト表示する。

### 4.6 右ペイン（フォルダ内容リスト）
- 左ペインで選択中のフォルダの内容（サブフォルダ＋ファイル）を一覧表示する。
- 表示カラム: アイコン、名前、種類（フォルダ/ファイル）、サイズ、更新日時。
- サブフォルダは一覧の上部にまとめて表示し、ファイルをその下に表示する。
- **シングルクリック**で対象を選択状態にする。選択済みの項目を再度クリックすると選択解除する。
- 複数選択は不可とし、常に1項目のみの選択とする。
- フォルダをクリックした場合も選択のみとし、ダブルクリックによるフォルダ移動は実装しない。フォルダへの移動は左ペインのツリー操作またはアドレスバーで行う。
- ファイルを選択した場合の追加動作（プレビュー等）はなし。
- 選択中の項目はハイライト表示し、ツールバーのボタン活性状態を更新する。

## 5. ファイル操作仕様

### 5.1 コピー・貼り付け
- コピー操作でファイル・フォルダのパスとバックエンド情報を内部クリップボードに記録する。
- 貼り付け先は、その時点で右ペインに表示中のフォルダとする（左ペインや右ペインでの別途選択は不要）。
- バックエンド間をまたぐコピーに対応する（例: disk のファイルを gdrive にコピー）。
- フォルダのコピーは再帰的に実行する。

### 5.2 移動（切り取り＋貼り付け）
- 切り取り操作でファイル・フォルダのパスとバックエンド情報を内部クリップボードに記録する。
- 貼り付け実行時に以下の順序で処理する:
  1. 貼り付け先に同名ファイルが存在しないことを確認する（存在する場合は確認ダイアログを表示）。
  2. 貼り付け先へコピーを実行する。
  3. コピー完了後、コピー先のファイルが正常に存在することをverifyする。
  4. verifyが成功した場合のみ、元ファイルを削除する。
  5. verifyが失敗した場合はエラーメッセージを表示し、元ファイルは削除しない。
- バックエンド間をまたぐ移動に対応する。

### 5.3 削除
- 削除前に確認ダイアログを表示する（「{ファイル名} を削除しますか？」）。
- momo_gdrive の削除はゴミ箱への移動（trash）とする。
- momo_disk の削除はゴミ箱への移動とする（`force=False` で呼び出す）。
- momo_github への削除操作はエラーダイアログを表示して操作を中止する。

### 5.4 新規フォルダ作成
- フォルダ名の入力ダイアログを表示する。
- 現在右ペインに表示中のフォルダ内に作成する。
- momo_github への新規フォルダ作成はエラーダイアログを表示して操作を中止する。

### 5.5 github バックエンドへの書き込み制限
以下の操作を github バックエンドに対して実行しようとした場合、操作を中止してエラーダイアログを表示する。エラーメッセージ例: `「GitHub フォルダは読み取り専用です。書き込みはできません。」`
- 切り取り（移動元が github の場合）
- 貼り付け（貼り付け先が github の場合）
- 削除
- 新規フォルダ作成

## 6. エラーハンドリング
| エラー状況 | 表示方法 | 挙動 |
|---|---|---|
| github への書き込み操作 | エラーダイアログ | 操作中止 |
| 存在しないアドレスへのジャンプ | アドレスバー下にインラインメッセージ | 移動しない |
| 移動時のverify失敗 | エラーダイアログ | 元ファイルを保持したまま中止 |
| 貼り付け先に同名ファイルが存在 | 確認ダイアログ（上書き/キャンセル） | ユーザー選択に従う |
| disk 未接続（ディレクトリ未選択） | ツリー展開時に自動でダイアログ表示 | 選択完了後に展開再試行 |
| disk 未接続でアドレスバーからジャンプ | アドレスバー下にインラインメッセージ | 移動しない |
| disk 初期化失敗（py ロードエラー） | エラーダイアログ | 操作中止 |
| gdrive クリック | 「実装中」情報ダイアログ | 操作中止（スタブ） |
| gdrive アドレスバーからジャンプ | アドレスバー下にインラインメッセージ | 移動しない |
| ネットワークエラー | エラーダイアログ | 操作中止。キャッシュがあれば表示を維持 |

## 7. 状態管理
- **内部クリップボード**: コピー・切り取り操作の対象パス、バックエンド情報、操作種別（copy/cut）を保持する。貼り付け完了後にクリアする（cut の場合のみ）。
- **選択状態**: 右ペインで現在選択中の項目のパスとバックエンド情報を保持する。フォルダ移動時にクリアする。
- **各バックエンドの接続状態**: `disconnected` / `connecting` / `ready` / `connected` / `error` の各状態を管理し、ツリーのルートノードバッジに反映する。
- **disk のルート名**: 接続後に `rootName` として保持し、ステータスバーの `#sb-disk` エリアに表示する。

## 8. ユーザー体験（UX）設計指針
- **操作中のフィードバック**: API通信中はツールバーボタンを非活性にし、右ペインにローディングインジケータを表示する。
- **バックエンドの識別**: 左ペインのルートノードおよびアドレスバーのプレフィックスに、バックエンドごとに異なるアイコンとカラーを付与して視覚的に区別する（github: 🐙 緑、disk: 💾 黄、gdrive: 📱 赤）。アドレスバーの入力テキスト色もバックエンドに連動して変化する。
- **読み取り専用の明示**: github フォルダ配下を表示中の場合、ツールバーの書き込み系ボタン（切り取り・削除・新規フォルダ）をグレーアウトして操作不可を視覚的に示す。
- **パンくずとアドレスバーの同期**: 左ペインのツリー選択・アドレスバー直接入力のどちらで移動した場合も、アドレスバーと左ペインのハイライトを常に同期する。
- **disk 接続状態の明示**: disk 接続後はステータスバーに選択中のルートフォルダ名を常時表示する。
